# Implementation Plan: Siri Voice Access

**Branch**: `032-siri-voice-access` | **Date**: 2026-03-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/032-siri-voice-access/spec.md`

## Summary

Add hands-free voice access to the family assistant via Apple Shortcuts and Siri. A new `/api/v1/voice/message` endpoint accepts natural language text from Siri, processes it through the existing assistant pipeline, and returns voice-optimized responses. Each parent gets per-user Bearer tokens for authentication and identity. Preset Shortcuts provide one-phrase access to common tasks (calendar, grocery, dinner, reminders). A hybrid sync/async pattern handles the ~25-second Apple Shortcuts timeout — fast responses return immediately, slow ones acknowledge and fall back to WhatsApp delivery.

## Technical Context

**Language/Version**: Python 3.12 (existing codebase)
**Primary Dependencies**: FastAPI, anthropic SDK, existing deps — no new Python dependencies
**Storage**: JSON files in `data/` (existing pattern for conversation logs)
**Testing**: pytest (existing)
**Target Platform**: Railway cloud server (existing) + iOS 15+ (Apple Shortcuts client)
**Project Type**: Web service (existing FastAPI app) + Apple Shortcuts (client-side, no code)
**Performance Goals**: Response within 18 seconds for sync path; acknowledgment within 2 seconds for async fallback
**Constraints**: ~25-second Shortcuts HTTP timeout (non-configurable); single uvicorn worker (APScheduler requirement); cannot read HTTP status codes from Shortcuts
**Scale/Scope**: 2 users, ~20 voice interactions/day
