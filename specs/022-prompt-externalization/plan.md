# Implementation Plan: Prompt Externalization

**Branch**: `022-prompt-externalization` | **Date**: 2026-03-08 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/022-prompt-externalization/spec.md`

## Summary

Move all prompt text from Python string literals to external Markdown files, organized in `src/prompts/`. The system prompt (~376 lines) splits into 8 composable section files; 71 tool descriptions split into 12 module-grouped files; 11 dynamic classification/generation prompts become template files with `{placeholder}` syntax. A lightweight loader module (`src/prompts.py`) reads everything at startup and caches in memory. Zero new dependencies вЂ” stdlib `pathlib` and `str.format()` only.

## Technical Context

**Language/Version**: Python 3.12 (existing codebase)
**Primary Dependencies**: None new вЂ” uses `pathlib`, `functools.lru_cache`, `str.format()` (all stdlib)
**Storage**: Markdown files in `src/prompts/` directory (committed to git, included in Docker image)
**Testing**: pytest (existing CI pipeline вЂ” smoke test + startup validation)
**Target Platform**: Linux server (Docker on NUC + Railway)
**Project Type**: Web service (FastAPI)
**Performance Goals**: Startup time increase < 100ms (reading ~30 small files)
**Constraints**: Zero functional regression вЂ” assistant behavior must be identical before and after
**Scale/Scope**: 31 prompt files total (8 system + 12 tool + 11 template)

## Key Design Decisions

### 1. System Prompt Assembly

```
src/prompts/system/01-identity.md + 02-response-rules.md + ... + 08-advanced.md
в†’ concatenated with "\n\n" separator
в†’ returned as single string
в†’ used by handle_message() exactly as before (date sandwich + preferences appended)
```

The existing runtime composition (date line + system prompt + date line + preferences) stays unchanged. Only the source of the static SYSTEM_PROMPT content changes.

### 2. Tool Description Merging

Each tool description file uses `## tool_name` Markdown headers:

```markdown
## get_calendar_events

Fetch upcoming events from Google Calendar. Returns events for...

## get_outlook_events

Returns Jason's Cisco work calendar events...
```

The loader parses these into a `dict[str, str]` mapping tool names to descriptions. The TOOLS array in `assistant.py` is then built by merging:
- **Description**: from `load_tool_descriptions()["tool_name"]`
- **Schema**: from Python dict (stays in `assistant.py` or moves to a `TOOL_SCHEMAS` dict)

### 3. Template Rendering

Templates use Python `.format()` placeholders:

```markdown
Classify this Amazon item: "{item_title}" (${item_price})

Given the family's YNAB budget categories:
{category_list}
```

Called via:
```python
from src.prompts import render_template
prompt = render_template("amazon_classification", item_title=title, item_price=price, ...)
```

### 4. Startup Validation

On import of `src/prompts.py`:
- Verify `src/prompts/system/` has at least one `.md` file
- Verify all expected tool names have descriptions (warn on missing, don't crash)
- Verify all template files referenced in code exist

### 5. Docker Compatibility

No Dockerfile changes needed вЂ” `src/prompts/` is already included via `COPY src/ ./src/` (or equivalent). Prompt files ship with the image.
